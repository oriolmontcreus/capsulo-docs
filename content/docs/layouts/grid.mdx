The `Grid` layout is a responsive container that organizes fields into columns. It supports mobile-first responsive design with configurable columns and gaps for different screen sizes.

## Usage

```typescript
import { Grid } from "@/lib/form-builder/layouts/Grid/grid.builder";
```

```typescript
// [!code word:Grid]
Grid({ base: 1, md: 2 }).contains([
  Input("firstName").label("First Name"),
  Input("lastName").label("Last Name"),
]);
```

<TypeTable

      description: "Adds fields to be rendered inside the grid container.",

The responsive value object allows different values for each breakpoint:

## Column Spanning

Fields inside a grid can span multiple columns using the `colSpan()` method. This is applied directly to the field builder for a more intuitive API.

### colSpan Method

<TypeTable
  type={{
    "Field.colSpan(value)": {
      description:
        "Sets how many columns this field should span in a grid layout. Supports numbers, 'full' for all columns, or responsive objects.",
      type: "number | 'full' | ResponsiveValue",
    },
  }}
/>

### colSpan Values

- **number**: Span that many columns on all breakpoints (e.g., `colSpan(2)`)
- **"full"**: Span all available columns (e.g., `colSpan("full")`)
- **ResponsiveValue**: Different spans per breakpoint (e.g., `colSpan({ base: 1, md: 2 })`)

If a colSpan exceeds the available columns, it automatically caps at the maximum available.
