A Capsule is a self-contained folder that holds everything related to a content component: the schema, the UI component, and the auto-generated TypeScript types.

## Anatomy of a Capsule

Take a Hero section as an example. You create a `hero/` folder with:

<Files>
  <Folder name="hero" defaultOpen>
    <File name="hero.schema.tsx" />
    <File name="Hero.astro" />
    <File name="hero.types.ts" />
  </Folder>
</Files>

- `hero.schema.tsx` — defines the editable fields and their structure
- `Hero.astro` — the component responsible for rendering it. This could also be `Hero.tsx`, `Hero.vue`, `Hero.svelte`, or any other [frontend framework supported by Astro](https://docs.astro.build/it/guides/framework-components/)
- `hero.types.ts` — automatically generated by Capsulo's schema watcher. No manual work needed.

<br/>
<Steps>

<Step>

### Define the schema

`hero.schema.tsx` defines every editable field for this component using the Fluent API:

```tsx
import { createSchema } from "@/lib/form-builder/builders/SchemaBuilder";
import { Input, Textarea } from "@/lib/form-builder/fields";

export const HeroSchema = createSchema(
  "Hero",
  [
    Input("title")
      .label("Hero title")
      .description("The main headline")
      .required()
      .translatable()
      .defaultValue("Welcome to Capsulo"),

    Textarea("subtitle")
      .label("Subtitle")
      .rows(3)
      .translatable()
      .defaultValue("A content management system for developers"),
  ],
  "Hero section with title and CTA",
  "hero",
);
```

</Step>

<Step>

### Build the component

`Hero.astro` — or `Hero.tsx`, `Hero.vue`, `Hero.svelte`, or any other [frontend framework supported by Astro](https://docs.astro.build/it/guides/framework-components/):

<Tabs items={['Astro', 'React', 'Vue']}>

<Tab value="Astro">

```astro
---
import type { SchemaProps } from '@/lib/schema-props';
import { HeroSchema } from './hero.schema';

export type Props = SchemaProps<typeof HeroSchema>;
const { title, subtitle } = Astro.props;
---

<section>
  <h1>{title}</h1>
  <p>{subtitle}</p>
</section>
```

</Tab>

<Tab value="React">

```tsx
import type { SchemaProps } from "@/lib/schema-props";
import { HeroSchema } from "./hero.schema";

type Props = SchemaProps<typeof HeroSchema>;

export function Hero({ title, subtitle }: Props) {
  return (
    <section>
      <h1>{title}</h1>
      <p>{subtitle}</p>
    </section>
  );
}
```

</Tab>

<Tab value="Vue">

```vue
<script setup lang="ts">
import type { SchemaProps } from "@/lib/schema-props";
import { HeroSchema } from "./hero.schema";

defineProps<SchemaProps<typeof HeroSchema>>();
</script>

<template>
  <section>
    <h1>{{ title }}</h1>
    <p>{{ subtitle }}</p>
  </section>
</template>
```

</Tab>

</Tabs>

</Step>

<Step>

### Get types automatically

`hero.types.ts` is automatically generated by Capsulo's schema watcher

```typescript
// hero.types.ts — auto-generated, do not edit
export interface HeroTypes {
  title: string; // required Input → string
  subtitle?: string; // optional Textarea → string | undefined
}
```

Add a toggle field and Capsulo knows it's a `boolean`. A required input becomes a `string`. An optional one becomes `string | undefined`. Your schema _is_ your type definition

</Step>

</Steps>

<Callout type="info" title="Schema watcher">
  The schema watcher runs automatically when you start your development server.
  It monitors your capsule folders and regenerates `.types.ts` files
  automatically whenever you update a schema.
</Callout>

## Auto discovery

When you import a Capsule component into one of your pages, Capsulo detects it automatically and surfaces the right CMS fields for that page — no registration, no configuration, no "hey CMS, this page uses this component."

```astro
---
// src/pages/index.astro
import Hero from '@/capsules/hero/Hero.astro';
import Features from '@/capsules/features/Features.astro';
---

<Hero />
<Features />
```

Visit `/admin` and the CMS fields for `Hero` and `Features` are already there, scoped to the index page. You just build your website normally, and Capsulo figures the rest out.

One folder. A schema, a component, and types that write themselves. That's a Capsule.
