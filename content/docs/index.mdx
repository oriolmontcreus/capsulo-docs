## What are Schemas?

Schemas are the core of Capsulo. They define the structure of your content components using code, creating a type-safe bridge between your CMS interface and your website components.

<Steps>

<Step>

### Define Your Schema

Create a schema using React/TSX and the Fluent API:

```typescript
export const HeroSchema = createSchema(
  'Hero',
  [
    Input('title')
      .label('Hero Title')
      .required()
      .defaultValue('Welcome'),
    
    Grid({ base: 1, md: 2 })
      .contains([
        Input('ctaButton').label('CTA Button'),
        Input('ctaLink').label('CTA Link')
      ])
  ],
  'Hero section with title and CTA',
  'hero'
);
```

This single schema automatically:
- Creates an editable form in the CMS interface
- Generates TypeScript types for your components
- Provides runtime validation with Zod
- Sets default values for all fields

</Step>

<Step>

### Use in Your Component

Use the schema data in your Astro components with full type safety:

```astro
---
import { getSchemaProps } from '@/lib/schema-props';
import { HeroSchema } from '@/lib/form-builder/schemas/hero.schema';
import type { SchemaProps } from '@/lib/schema-props';

export type Props = SchemaProps<typeof HeroSchema>;
const props = getSchemaProps(HeroSchema, Astro.props);
const { title, ctaButton, ctaLink } = props;
---

<section>
  <h1>{title}</h1>
  <a href={ctaLink}>{ctaButton}</a>
</section>
```

</Step>

<Step>

### Edit in the CMS

Content creators can now edit the Hero component directly in the CMS interface at `/admin` with a beautiful, auto-generated form. No code required!

</Step>

</Steps>
